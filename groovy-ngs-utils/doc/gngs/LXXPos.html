Package: gngs

[Java] Class LXXPos

    • gngs.LXXPos

    • LXXPos models genomic positions including alt and random contigs into a flat space fitting into 63bit unsigned Java long type.

      The positions computed by LXXPos are ordered such that within any given chromosome or contig, the positions are ordered numerically. Ordering between chromosomes / contigs is lexical. For example, chr1 precedes chr10, which precedes chr2. However the ordering of alt contigs is also lexical, so, for example, chr1_KI270189 comes after chr19.

      This class uses a complex strategy to pack the description of the genomic coordinates into 63 bits. The bits are used different ways when an alt or random contig is represented to when a primary contig is. This is because the primary contigs are much bigger and hence need more bits to represent the position within the contig, while the alternate contigs are small but numerous, so more bits are needed to represent which alternate contig is selected.

      The structure for an alt contig is like so: 

         primary    alt contig
   contig      type           alt contig number                       position
 |--5 bits--|--4 bits--|-------------26 bits----------------|---------------25 bits----------------|
      The alt contig representation is distinguished by the first bit of the alt contig type being 1. When this bit is zero, the structure is assumed to represent a primary contig coordinate, represented as follows: 
       |--5 bits--|--4 bits--|-----------------------53 bits---------------------------------------------|
      This allows a precision of billions of bases on the primary contigs, and millions of bases on the alt contigs.
